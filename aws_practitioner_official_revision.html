Exam Blueprint

Official Principles and Core Ideas

These topic blocks are based directly on the four official CLF-C02 domain pages. Each task includes a short study description plus the main knowledge and skill points AWS calls out.

Content Domain 1: Cloud Concepts

Task Statement 1.1: Define the benefits of the AWS Cloud

The value proposition of AWS connects cloud adoption with global infrastructure, speed of deployment, global reach, high availability, elasticity, and agility.

Task Statement 1.2: Identify design principles of the AWS Cloud

The AWS Well-Architected Framework is organized around six pillars: Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization, and Sustainability. Each pillar highlights a different design priority and together they form AWS’s main architecture reference model.

Task Statement 1.3: Understand the benefits of and strategies for migration to the AWS Cloud

Cloud adoption strategies and resources that support migration include the AWS Cloud Adoption Framework (AWS CAF), which organizes cloud adoption across business, people, governance, platform, security, and operations perspectives. Migration approaches include database replication and offline transfer with AWS Snowball.

Task Statement 1.4: Understand concepts of cloud economics

Cloud economics includes fixed versus variable costs, on-premises cost considerations, licensing strategies such as Bring Your Own License (BYOL) versus included licenses, rightsizing, benefits of automation, and economies of scale.

Content Domain 2: Security and Compliance

Task Statement 2.1: Understand the AWS shared responsibility model

The shared responsibility model divides security into AWS responsibilities for the cloud and customer responsibilities in the cloud. Customers manage items like identities, data, and configuration, while AWS manages the underlying infrastructure. The balance changes with services such as Amazon RDS, AWS Lambda, and Amazon EC2.

Task Statement 2.2: Understand AWS Cloud security, governance, and compliance concepts

Compliance and governance concepts, benefits of cloud security such as encryption, and where security logs are captured form the core of this task. Key ideas include where compliance information is published, how customers secure resources, what encryption in transit and encryption at rest mean, and which services support monitoring, auditing, governance, and reporting.

Task Statement 2.3: Identify AWS access management capabilities

Identity and access management includes AWS IAM, the importance of protecting the root user, the principle of least privilege, and AWS IAM Identity Center. Related points include access keys, password policies, secrets storage, multi-factor authentication (MFA), cross-account roles, groups, custom and managed policies, root-user-only tasks, and federated identity concepts.

Task Statement 2.4: Identify components and resources for security

Security capabilities and security documentation provided by AWS include services such as AWS WAF, AWS Firewall Manager, AWS Shield, Amazon GuardDuty, and AWS Trusted Advisor, along with security information published through the Knowledge Center and AWS documentation sources.

Content Domain 3: Cloud Technology and Services

Task Statement 3.1: Define methods of deploying and operating in the AWS Cloud

Ways of provisioning and operating in AWS, ways to access AWS services, and deployment models include the AWS Management Console, APIs, software development kits (SDKs), AWS CLI, and infrastructure as code (IaC), along with cloud, hybrid, and on-premises deployment models.

Task Statement 3.2: Define the AWS global infrastructure

This task includes Regions, Availability Zones, and edge locations. A Region is a separate geographic area, an Availability Zone is an isolated location inside a Region, and an edge location helps deliver content closer to users. Together they support high availability, low latency, disaster recovery, and data sovereignty.

Task Statement 3.3: Identify AWS compute services

Compute services include suitable Amazon EC2 instance types, container services such as Amazon ECS and Amazon EKS, serverless compute such as AWS Fargate and AWS Lambda, Auto Scaling for elasticity, and load balancers for traffic distribution.

Task Statement 3.4: Identify AWS database services

Database services and database migration include the distinction between EC2-hosted databases and managed databases. SQL databases include Amazon RDS and Amazon Aurora. NoSQL databases include Amazon DynamoDB. Memory-based databases include Amazon ElastiCache, and migration tools include AWS DMS and AWS SCT.

Task Statement 3.5: Identify AWS network services

Network services include virtual private cloud (VPC) components such as subnets and gateways, VPC security concepts such as network ACLs and security groups, the role of Amazon Route 53, and connectivity options such as AWS VPN and AWS Direct Connect.

Task Statement 3.6: Identify AWS storage services

Storage services include use cases for object storage, differences among Amazon S3 storage classes, block storage such as Amazon EBS and instance store, file services such as Amazon EFS and Amazon FSx, cached and hybrid storage options such as AWS Storage Gateway, along with lifecycle policies and AWS Backup.

Task Statement 3.7: Identify AWS AI/ML services and analytics services

AI/ML services and analytics services include the tasks performed by services such as Amazon SageMaker AI, Amazon Lex, and Amazon Kendra, along with analytics services such as Amazon Athena, Amazon Kinesis, AWS Glue, and Amazon QuickSight.

Task Statement 3.8: Identify services from other in-scope AWS service categories

This task includes additional in-scope categories such as application integration, business applications, customer enablement, developer tools, end-user computing, frontend web and mobile, and Internet of Things (IoT). The core idea is matching the right service to messaging, alerts, support, development, frontend delivery, end-user access, and IoT device management.

Content Domain 4: Billing, Pricing, and Support

Task Statement 4.1: Compare AWS pricing models

Compute purchasing options and storage pricing options include On-Demand Instances for short-term flexibility, Reserved Instances for steady workloads, Spot Instances for interruption-tolerant workloads, AWS Savings Plans for discounted usage commitments, Dedicated Hosts and Dedicated Instances for isolation needs, and Capacity Reservations for guaranteed EC2 capacity. Related pricing points include Reserved Instance flexibility, how Reserved Instances behave in AWS Organizations, data transfer pricing, and differences among storage tiers.

Task Statement 4.2: Understand resources for billing, budget, and cost management

Billing information, pricing information, AWS Organizations, and cost allocation tags form the core of this task. Important tools include AWS Budgets, AWS Cost Explorer, and AWS Pricing Calculator, along with consolidated billing and the role of cost allocation tags in reports such as the AWS Cost and Usage Report.

Task Statement 4.3: Identify AWS technical resources and AWS Support options

Official AWS documentation and resources, AWS Support plans, AWS Support Center, the AWS Partner Network, and AWS experts define this task. Relevant resources include whitepapers, blogs, documentation, AWS Prescriptive Guidance, AWS Knowledge Center, and AWS re:Post. Support options include Developer Support with technical guidance for development and test, Business Support with 24/7 access to AWS experts and the full set of AWS Trusted Advisor checks, Enterprise On-Ramp Support with a Technical Account Manager (TAM) for growing business-critical workloads, and Enterprise Support with a designated TAM and the most proactive guidance for enterprise environments. Current AWS Support pages emphasize account and billing support, AWS Trusted Advisor, and TAM guidance as key plan differences. This task also includes the roles of AWS Support, AWS Trusted Advisor, AWS Health Dashboard, AWS Health API, AWS Marketplace, AWS Professional Services, and AWS Solutions Architects.

Revision Focus

Exam Topics and Common Question Patterns

Based on the official CLF-C02 scope and the patterns that appear frequently in public practice-question sets, these are the topics that tend to come up again and again in revision and mock exams.

High-Frequency Topics

  • Shared responsibility model: AWS secures the cloud; the customer secures what runs in the cloud, such as identities, data, operating systems on Amazon EC2, and configurations.
  • IAM, MFA, and root user: the best answer is usually use least privilege, enable MFA, and do not use the root user for everyday tasks.
  • Regions and Availability Zones: a Region is a geographic area, an Availability Zone is an isolated location inside a Region, and using multiple Availability Zones improves high availability.
  • Pricing and support plans: On-Demand is for flexibility, Reserved and Savings Plans reduce cost for steady usage, Spot is cheapest for interruptible workloads, and higher support plans add faster help and more guidance.
  • Storage and databases: Amazon S3 is object storage, Amazon EBS is block storage for Amazon EC2, Amazon EFS is shared file storage, Amazon RDS is managed SQL, and Amazon DynamoDB is NoSQL.
  • Answer-style mini blocks: object storage = Amazon S3, managed relational database = Amazon RDS, NoSQL = Amazon DynamoDB, serverless code = AWS Lambda, audit API activity = AWS CloudTrail, metrics and alarms = Amazon CloudWatch, multiple account governance = AWS Organizations, and compliance reports = AWS Artifact.

Common Question Patterns

  • Best service for a use case: if the question says object storage, answer Amazon S3; if it says managed relational database, answer Amazon RDS; if it says serverless code, answer AWS Lambda.
  • Service comparisons: Amazon SNS pushes messages to many subscribers, Amazon SQS stores messages in a queue, AWS CloudTrail records API activity, and Amazon CloudWatch monitors metrics, logs, and alarms.
  • Difference-focused exam notes: Amazon S3 stores objects, Amazon EBS provides block storage for Amazon EC2, and Amazon EFS provides shared file storage; Amazon RDS is SQL and Amazon DynamoDB is NoSQL; AWS WAF filters web requests and AWS Shield protects against DDoS attacks; AWS Organizations manages accounts and AWS Control Tower sets up and governs a multi-account environment.
  • Cost optimization choices: for steady workloads, choose Reserved Instances or Savings Plans; for flexible short-term workloads, choose On-Demand; for interruptible workloads, choose Spot Instances.
  • Security responsibility: AWS manages physical infrastructure, while the customer manages users, permissions, operating systems on Amazon EC2, application settings, and data protections.
  • Support and governance: AWS Trusted Advisor gives recommendations, AWS Artifact provides compliance reports, AWS Organizations manages multiple accounts, and higher AWS Support plans add expert help and operational guidance.
